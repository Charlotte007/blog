块级格式化上下文（Block Fromatting Context,BFC）

定义

块格式化上下文是指一个独立的块级渲染区域，只有Block-level Box参与，该区域拥有一套渲染规则来约束块级盒子的布局，且与区域外部无关。
BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。

  • Block-level元素包含一下几种形式
    • block
    • list-item
    • table
    • table-*
    • flex
    • position (position 为 absolute 或 fixed)
    • float不为none

如何创建BFC

  • 根元素或包含根元素的元素
  • 浮动元素（元素的 float 不是 none）
  • 绝对定位元素（元素的 position 为 absolute 或 fixed）
  • 行内块元素（元素的 display 为 inline-block）
  • 表格单元格（元素的 display为 table-cell，HTML表格单元格默认为该值）
  • 表格标题（元素的 display 为 table-caption，HTML表格标题默认为该值）
  • 匿名表格单元格元素（元素的 display为 table、table-row、 table-row-group、table-header-group、table-footer-group（分别是HTML table、row、tbody、thead、tfoot的默认属性）或 inline-table）
  • overflow 值不为 visible 的块元素
  • 弹性元素（display为 flex 或 inline-flex元素的直接子元素）
  • 网格元素（display为 grid 或 inline-grid 元素的直接子元素）

BFC在布局中的应用

  • 两个相邻Box垂直方向margin重叠（margin-top/margin-bottom）
  • 相邻Box水平方向margin重叠（margin-lefrt/margin-right）
  • margin-top/margin-bottom导致父级坍塌
  • 清除内部浮动
  • 使用BFC避免文字环绕
  • 在多列布局中使用BFC
    • margin自适应两栏布局

CSS布局单位中百分比的使用 （%）

属性 相对于谁的哪个属性
width 基于父级的width
height 基于父级的height
margin （top,right,bottom,left 全部基于父级的width
padding（top,right,bottom,left） 全部基于父级的width
(left,right),(top,bottom) 基于(父元素/距离最近拥有position 非static的属性)的(width),(height);
font-size 基于继承得到的font-size
line-height 基于当前字体的font-size
translate(left,right) ,(top,bottom) 基于自身元素的(width),(height)
background-postion (left,right) ,(top,bottom) 基于容器剩余的(width),(height)计算
vertical-align 基于line-height
