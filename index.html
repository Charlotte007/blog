ES6模块概述

  • ES6 在语言标准的层面上，实现了模块功能，而且实现得相当简单，旨在成为浏览器和服务器通用的模块解决方案。
  • ES6 模块的设计思想是尽量的静态化，使得编译时就能确定模块的依赖关系，以及输入和输出的变量。
    这种加载称为“编译时加载”或者静态加载(静态加载就是编译时加载)，即 ES6 可以在编译时就完成模块加载，效率要比 CommonJS 模块的加载方式高(相比较于‘运行时加载’)。当然，这也导致了没法引用 ES6 模块本身，因为它不是对象。
  • ES6 模块不是对象，而是通过export命令显式指定输出的代码，再通过import命令输入。

ES6模块用法–export 命令

export导出的标识符,用于规定模块的对外接口

export {var1,var2,var3 }使用大括号指定索要输出的一组变量

export 输出函数或者类（class）

接口名需要与模块内部变量之间，建立了一一对应的关系

export命令可以出现在模块的任何位置，但只能是顶级作用域，如果处于块级作用域内，就会报错，不能处于 {}中
这是因为处于条件代码块之中，就没法做静态优化了，违背了 ES6 模块的设计初衷。

export语句输出的接口，与其对应的值是动态绑定关系，即通过该接口，可以取到模块内部实时的值

ES6模块用法–import 命令

import语句用于导入由另一个模块导出的绑定。

import大括号里面的变量名，必须与被导入模块对外接口的名称相同

import的常见用法

import命令具有提升效果，会提升到整个模块的头部，首先执行

import是静态执行，所以不能使用表达式和变量（同export 能是顶级作用域）
由于import是静态执行，所以不能使用表达式和变量，这些只有在运行时才能得到结果的语法结构

promise对象

发表于 2018-04-01

认识Pormise

1、Promise 是异步编程的一种解决方案，比传统的解决方案——回调函数和事件——更合理和更强大，Promise本身是一个构造函数，使用new 创建Promise对象；
2、Promise 简单说就是一个容器,里面保存着某个未来才会结束的事件（通常是一个异步操作）的结果
3、一个 Promise有以下几种状态:

  • pending: 初始状态，既不是成功，也不是失败状态。
  • fulfilled: 意味着操作成功完成。
  • rejected: 意味着操作失败。

pending 状态的 Promise 对象可能触发fulfilled 状态并传递一个值给相应的状态处理方法，也可能触发失败状态（rejected）并传递失败信息。当其中任一种情况出现时，Promise 对象的 then 方法绑定的处理方法（handlers ）就会被调用（then方法包含两个参数：onfulfilled 和 onrejected，它们都是 Function 类型。当Promise状态为fulfilled时，调用 then 的 onfulfilled 方法，当Promise状态为rejected时，调用 then 的 onrejected 方法， 所以在异步操作的完成和绑定处理方法之间不存在竞争）。

因为 Promise.prototype.then 和 Promise.prototype.catch 方法返回promise 对象， 所以它们可以被链式调用。

如果一个promise对象处在fulfilled或rejected状态而不是pending状态，那么它也可以被称为settled状态。常见的resolved ，它表示promise对象处于fulfilled状态，方便与 resolve(value)对应

4、Promise对象的特点：

  • 对象的状态不受外界影响。Promise对象代表一个异步操作，只有异步操作的结果，可以决定当前是哪一种状态，任何其他操作都无法改变这个状态
  • 一旦状态改变，就不会再变，任何时候都可以得到这个结果
  • 链式调用，Promise.prototype.then 和 Promise.prototype.catch 方法返回promise 对象

5、Promise也有一些缺点：

  • 无法取消Promise，一旦新建它就会立即执行，无法中途取消。
  • 如果不设置回调函数，Promise内部的错误不会反应到外部。
  • 当处于未完成态时，无法得知目前进展到哪一个阶段。

6、按照用途解释：

  • 主要用于异步计算
  • 可以将异步操作队列化，按照期望的顺序执行，返回符合预期的结果
  • 可以在对象之间传递和操作Promise，帮助我们处理队列

为什么会有promise？

  • 常见的异步操作，事件，回调函数（ajax fileAPI）
  • 回调地狱,难以维护

Pormise基本用法

Promise对象是一个构造函数，用来生成Promise实例，使用new创建promise的实例

Promise构造函数接受一个函数作为参数,该函数的两个参数分别是resolve和reject。它们是两个函数，由 JavaScript 引擎提供，不用自己部署。

resolve函数的作用是，将Promise对象的状态从“未完成”变为“成功”（即从 pending 变为 resolved），在异步操作成功时调用，并将异步操作的结果，作为参数传递出去；
reject函数的作用是，将Promise对象的状态从“未完成”变为“失败”（即从 pending 变为 rejected），在异步操作失败时调用，并将异步操作报出的错误，作为参数传递出去。

then方法可以接受两个回调函数作为参数。

第一个回调函数是Promise对象的状态变为resolved时调用;
第二个回调函数是Promise对象的状态变为rejected时调用。其中，第二个函数是可选的，不一定要提供。这两个函数都接受Promise对象传出的值作为参数。

Promise原型上常见的方法

  • Promise.prototype.then(onFulfilled, onRejected)
    添加解决(fulfillment)和拒绝(rejection)回调到当前 promise, 返回一个新的 promise, 将以回调的返回值来resolve.
    • 状态响应式函数，可以返回新的pormise，或者其它值 （resolve reject return 默认undefined）
    • 若果返回新的promise，那么下一级的then 会在新Promise状态改变后执行
    • 如果返回其他值，则立即会进入下一级的.then()

  • 因为.then() 返回的还是 Promise实例，
  • 会等到里面的 .then()执行完毕后，再执行外面的.then(),
  • 方法就是, 直接展开内部的 .then()按照顺序，链式组装.then()

  • Promise.prototype.catch()
    Promise.prototype.catch方法是.then(null, rejection)的别名，用于指定发生错误时的回调函数，返回一个新的 fulfilled的promise；
    一般都是将函数的错误处理交个catch统一处理，而不是reject；catch不仅可以捕获rejected状态的函数，还可以捕获throw error抛出的错误

    如果异步操作抛出错误，状态就会变为rejected，就会调用catch方法指定的回调函数，处理这个错误。另外，then方法指定的回调函数，如果运行中抛出错误，也会被catch方法捕获。
    一般来说，不要在then方法里面定义 Reject 状态的回调函数（即then的第二个参数），总是使用catch方法。

  • Promise.prototype.finally(onFinally)
    添加一个事件处理回调于当前promise对象，并且在原promise对象解析完毕后，返回一个新的promise对象。回调会在当前promise运行完毕后被调用，无论当前promise的状态是完成(fulfilled)还是失败(rejected)，该方法是 ES2018 引入标准的。

任务队列

JavaScript语言的一大特点就是单线程，也就是说，同一个时间只能做一件事,这样能提高效率;单线程就意味着，所有任务需要排队，前一个任务结束，才会执行后一个任务。如果前一个任务耗时很长，后一个任务就不得不一直等着。

任务可以分成两种，一种是同步任务（synchronous），另一种是异步任务（asynchronous）。同步任务指的是，在主线程上排队执行的任务，只有前一个任务执行完毕，才能执行后一个任务；异步任务指的是，不进入主线程、而进入”任务队列”（task queue）的任务，只有"任务队列"通知主线程，某个异步任务可以执行了，该任务才会进入主线程执行。

异步执行的运行机制如下:（同步执行也是如此，因为它可以被视为没有异步任务的异步执行。）

  1. 所有同步任务都在主线程上执行，形成一个执行栈（execution context stack）。
  2. 主线程之外，还存在一个”任务队列”（task queue）。只要异步任务有了运行结果，就在”任务队列”之中放置一个事件。
  3. 一旦”执行栈”中的所有同步任务执行完毕，系统就会读取”任务队列”，看看里面有哪些事件。那些对应的异步任务，于是结束等待状态，进入执行栈，开始执行。
  4. 只要主线程空了，主线程不断重复上面的第三步，读取”任务队列”；

异步任务的分类

异步任务可以分为两类，macrotask(宏任务)，microtask(微任务)

macrotask 包含：script(整体代码)、setTimeout、setInterval、I/O、UI交互事件、postMessage 、requestAnimationFrame 、MessageChannel、etImmediate(Node.js 环境)
microtask 包含：Promise.then、MutaionObserver、process.nextTick(Node.js 环境)

Event Loop

主线程从"任务队列"中读取事件，这个过程是循环不断的，所以整个的这种运行机制又称为Event Loop（事件循环）。

事件循环基于任务队列，EventLoop中，可以有多个任务队列，任务队列是任务(task)的集合，每一个任务都拥有一个任务源(task source), 同一个task source(任务源)必须放在同一个task queue(任务队列)中。

事件循环中，每进行一次循环称为tick。每一次tick处理的方式规则如下：

  1. 每一次tick(事件循环)对于任务队列是先进先出（执行最先进入任务队列的任务，所以称为队列）。

  2. 每一次实践循环结束后，会优先检查microtasks(微任务)，如果存在会不断执行，清空microtasks queue。

定时器

注意：setTimeout，setIntervar第二个参数时间的具体含义指的是，等待进入macrotask（队列的时间），而不是等待执行的时间🕙。如果主线程的执行，发生了长时间的阻塞，那么可能会照成一种结果就是两个不同等待时间的setTimeout可能会在阻塞之后，执行会紧挨在一起

将setTimeout插入macrotask queue中，setTimeout的延迟时间指的是，setTimeout插入macrotask queue的延迟时间

for ( var i = 0;i<5; i++) {
    console.log(i);
}// 直接输出为0,1,2,3,4

for( var i = 0;i<5;i++) {
    setTimeout(function() {
        console.log(i);
    },1000*i)
} // 先输出一个 5，再每个一秒输出剩下的4个 5

for( var i = 0;i<5;i++) {
    (function(i){
        setTimeout(function () {
            console.log(i);
        },1000)
    })(i);
}// 每隔一秒输出i

原型的定义

每个函数都有一个prototype的属性，他是一个指针，指向一个对象，这个对象中包含特定类型的所有实例共享的属性和方法;简而言之就是：每个函数自带的包含特定类型的所有实例共享的属性和方法的对象；

原型的实例说明

函数自带的包含特定类型的所有实例共享的属性和方法的对象；

理解原型对象 prototype

1、原型对象会在创建新函数的时候，就会为该函数创建一个prototype属性，指向这个函数的原型对象；默认情况下都会自动回去一个 constructor的属性，指向原型所在的函数；至于其他的方法，默认是从 Object继承而来；
2、ES5中可以通过 指针[[Prototype]],访问函数的原型，但是ES5中没有标准化，只有 FireFox，Safari，Chrome在每个对象上支持一个__proto__属性,用于实例访问构造函数的原型对象
需要明确的是，__proto__是链接实例和构造函数原型（prototype）的，而不是实例和构造函数之间

原型链的定义

原型链是作为继承的主要方法，基本思想就是利用原型让一个引用类型去继承另一个应用类型的属性和方法；就是一个构造函数（父）的实例是另一个构造函数（子）的原型，（SonType.prototype = new FatherType() => SonType.prototype.__proto__ === FatherType.prototype ）,通过层层递进 重写原型对象,实现继承；原型链是依赖于__proto__，而不是prototype。所以可以简单理解为：函数继承中，链接所有层级构造函数原型和实例的链条，称之为原型链

函数参数的默认值

ES6中函数参数的默认值的有点
1、代码简洁，可读性高，调用时可以省略传入参数
2、避免了参数转Boolean值为flase的情形，采用的是默认值的方式

使用默认参数时需要注意的事项

1、参数变量是默认声明的，所以不能用let或const再次声明，否则会报错
2、使用参数默认值时，函数不能有同名参数，否则会报错
3、参数默认值不是`传值`的,而是`每次`都`重新计算默认值表达式`的`值`。也就是说，参数默认值是惰性求值的。( *`简单说就是用到的时候再算`*);
4、通俗的来讲可以认为没有 参数都是默认值的，初始的默认值为 `undefined`
5、默认参数的位置 最好写在函数参数的最后位，方便调用时省略有默认值的参数

与解构赋值默认值结合使用

注意： 函数参数的默认值，注意 = 赋值的位置，实际函数参数值以解构赋值的结果为准
写法一函数参数的默认值是空对象，但是设置了对象解构赋值的默认值；
写法二函数参数的默认值是一个有具体属性的对象，但是没有设置对象解构赋值的默认值。

最简单的方法： 拿匹配的 传入参数 替换默认值的位置

rest 参数 （数组类型）

ES6 引入 rest 参数（形式为…变量名），用于获取函数的多余参数，这样就不需要使用arguments对象了（类数组：拥有length属性，不具有数组所具有的方法的对象）。rest 参数搭配的变量是一个数组，该变量将多余的参数放入数组中。
1、rest 参数是真正的数组，数组特有的方法都可以使用,不需要使用 Array.prototype.slice.call(arrayLike)或者[].slice.call(arrayLike) 将arguments类数组装换成真数组了
2、rest 参数是指，剩余的参数
3、rest 参数之后不能再有其他参数（即只能是最后一个参数），否则会报错

箭头函数

使用“箭头”（=>）定义函数。

箭头函数有几个使用注意点

（1）箭头函数没有自己的this，所以当然也就不能用call()、apply()、bind()这些方法去改变this的指向；
函数体内的this对象，就是定义时所在的对象，而不是使用时所在的对象。 箭头函数里面根本没有自己的this，而是引用外层的this(var _this = this)。

（2）不可以当作构造函数，也就是说，不可以使用new命令，否则会抛出一个错误。(因为它没有this)

（3）不可以使用arguments对象，该对象在函数体内不存在。如果要用，可以用 rest 参数代替。

（4）不可以使用yield命令，因此箭头函数不能用作 Generator 函数。

严格模式

ES6 的模块自动采用严格模式，不管你有没有在模块头部加上”use strict”;。

严格模式主要有以下限制。

  • 变量必须声明后再使用
  • 函数的参数不能有同名属性，否则报错
  • 不能使用with语句
  • 不能对只读属性赋值，否则报错
  • 不能使用前缀 0 表示八进制数，否则报错
  • 不能删除不可删除的属性，否则报错
  • 不能删除变量delete prop，会报错，只能删除属性delete global[prop]
  • eval不会在它的外层作用域引入变量
  • eval和arguments不能被重新赋值
  • arguments不会自动反映函数参数的变化
  • 不能使用arguments.callee
  • 不能使用arguments.caller
  • 禁止this指向全局对象
  • 不能使用fn.caller和fn.arguments获取函数调用的堆栈
  • 增加了保留字（比如protected、static和interface）

上面这些限制，模块都必须遵守。由于严格模式是 ES5 引入的，不属于 ES6，所以请参阅相关 ES5 书籍，本书不再详细介绍了。

其中，尤其需要注意this的限制。ES6 模块之中，顶层的this指向undefined，即不应该在顶层代码使用this。

模板字符串

模板字符串（template string）是增强版的字符串，用反引号（`）标识。它可以当作普通字符串使用，也可以用来定义多行字符串，或者在字符串中嵌入变量。

模板字符串中嵌入变量，需要将变量名写在${}之中。可以嵌入任意的 JavaScript 表达式，引用对象属性,调用函数，另一个模板字符串等
